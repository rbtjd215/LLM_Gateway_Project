### 슬라이드 #6-1: 시스템 프롬프트 엔지니어링 상세

**PPT 내용:**
- **좌측: Phase 2 — LLM Judge 시스템 프롬프트 핵심 구조**
  - BLOCKED 기준 6가지 명시: ① Prompt Injection ② Jailbreak ③ 번역·요약 경유 우회 ④ 자격증명 탈취 ⑤ 보안 프로토콜 비활성화 ⑥ 파괴적 명령어
  - 출력 포맷 강제: `{"intent": "SAFE"}` 또는 `{"intent": "BLOCKED", "reason": "..."}`
  - `Temperature = 0.0`, Fail-Closed 정책
- **우측: Phase 3 — Generative DLP 시스템 프롬프트 핵심 구조**
  - 기밀 카테고리 6가지: MATERIAL, PROCESS, PRICE, PERSON, CODE, DIMENSION

- **하단:** 동일 Qwen2.5:7b 모델에 별도 API 호출로 주입, `asyncio.gather()` 병렬 실행
- **[📸 캡처 삽입 권장]**: `security_core.py`의 `_JUDGE_SYSTEM_PROMPT`(L262~L277) 및 `_MASKING_SYSTEM_PROMPT`(L352~L368)

**대사 (1분 30초):**

### 슬라이드 #15: 향후 연구 방향

**PPT 내용:**
- **단기**: 70B 대형 모델 적용 + 양자화(Quantization) 등 로컬 경량화 기술 → 방어율 90%+
- **장기**: 산업별 특화 파인튜닝 (제조, 금융, 의료 등 도메인별 기밀 패턴 학습)

**대사 (40초):**
> "향후에는 70B 이상의 대형 모델과 양자화 경량화 기술을 적용하여 방어율을 90% 이상으로 끌어올리고, 산업별 특화 파인튜닝을 거쳐 각 산업의 고유한 기밀 패턴을 더 정밀하게 탐지하는 것을 목표로 합니다."

### 슬라이드 #16: 결론

**PPT 내용:**
- **한 줄 결론:** "설계도는 구조적으로 우수하다. 모델 고도화로 잠재력을 극대화할 수 있다."
- **핵심 수치 요약:** 정밀도 99.86% (한국어 100%), 방어율 약 2.9배 향상 (26.7%→78.33%), 사내 PC로 구동

**대사 (30초):**
> "정리하겠습니다. 오토코어 사내에서 외부 AI를 안전하게 사용하기 위한 보안 게이트웨이의 아키텍처를 설계하고, 7B 모델과 CPU만으로 그 작동을 검증했습니다. 설계가 올바르다는 것은 정밀도 99.86%가 말해줍니다. 더 큰 모델을 적용하면 방어율은 즉시 상승할 것입니다. 감사합니다."
