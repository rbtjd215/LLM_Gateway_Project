## ===== 발표자 A 파트 (슬라이드 1 ~ 13) =====

### 슬라이드 1: 표지
> "안녕하세요. 저희는 '로컬 LLM 기반 하이브리드 AI 보안 게이트웨이'를 주제로 연구를 진행한 팀입니다. 생성형 AI 시대에 기업의 기밀 정보를 보호하면서도 AI 활용을 안전하게 허용하는 보안 시스템을 설계하고 직접 구현했습니다. 저는 앞부분인 문제 정의와 아키텍처 설계, 시연을 발표할 발표자 A입니다."

### 슬라이드 2: AI 시대의 새로운 보안 위협
> "생성형 AI의 기업 도입이 확산되면서 새로운 보안 위협이 등장했습니다. Cyberhaven의 2024년 보고서에 따르면, 직원이 AI에 입력한 데이터 중 27.4%가 기밀 정보로 분류되었습니다."
> "실제로 2023년 삼성전자 반도체 부문에서는 ChatGPT에 소스코드를 입력하는 기밀 유출 사고가 발생하기도 했습니다. 또한 OWASP는 2025년 LLM 보안 위협 1위로 프롬프트 인젝션을 꼽았습니다. 즉, 현재 기업은 '비의도적 기밀 유출'과 '프롬프트 인젝션'이라는 두 가지 위협에 동시에 노출되어 있습니다."

### 슬라이드 3: 기존 보안 솔루션의 구조적 한계 분석
> "기존 솔루션들은 근본적인 한계가 있습니다. 첫째, 기존 DLP는 정형 데이터만 잡을 뿐, 문맥에 녹아든 비정형 기밀은 탐지하지 못합니다. 둘째, AI를 전면 차단하면 직원들은 개인 폰으로 우회하는 섀도우 AI 상태가 되어 통제가 불가능해집니다. 셋째, 클라우드 DLP는 검열을 위해 데이터를 외부로 보내야 해서 데이터 주권 문제가 생깁니다."
> "그래서 저희는 질문했습니다. 데이터 주권을 유지하면서, 기밀은 보호하되 AI 사용은 안전하게 허용하는 '중간 개입자'가 가능한가?"

### 슬라이드 4: AutoCore 보안 게이트웨이 핵심 설계 원칙
> "이를 위해 4가지 원칙을 세웠습니다. 첫째, 모든 검열을 사내 서버에서 처리하는 '완전한 로컬 처리'. 둘째, 정규식과 LLM을 결합한 '계층적 심층 방어'. 셋째, 기밀을 난수로 치환해 전송하고 응답 시 복원하여 UX를 보존하는 '동적 마스킹 및 역치환'. 넷째, AI 고유의 공격인 프롬프트 인젝션에 대응하기 위한 '결정론적 방어'입니다."

### 슬라이드 5: 3단계 하이브리드 보안 파이프라인
> "이 게이트웨이는 오토코어 사내 직원과 외부 AI 사이에 위치하는 보안 프록시입니다."
> "작동 순서입니다. 먼저 Phase 1에서 사번 같은 '정형 개인정보'를 정규식으로 초고속 마스킹합니다. 이 정규식을 통과한 텍스트에 대해, 이제 Phase 2와 Phase 3가 비동기 병렬로 실행됩니다. Phase 2는 로컬 LLM이 악의적 의도를 판독하고, Phase 3는 같은 LLM이 배합비처럼 문맥으로만 알 수 있는 '비정형 기밀'을 찾아 마스킹합니다."
> "안전하게 마스킹된 텍스트만 외부 AI에 전송되며, 외부 AI는 난수 토큰을 고유명사로 취급해 요약/번역을 수행합니다. 응답이 돌아오면 로컬 Redis를 통해 원본으로 완벽히 역치환하여 직원에게 전달합니다."

### 슬라이드 6: 핵심 검열 기술 구조: 판정(Judge)
> "Phase 2의 LLM 판정 로직입니다. 6대 차단 기준을 시스템 프롬프트로 명시하고, 반드시 SAFE나 BLOCKED 중 하나로만 JSON 응답을 하도록 강제했습니다."
> "특히 우측 코드를 보시면 Temperature를 0.0으로 고정하여 판정의 일관성을 확보했고, JSON 파싱 실패 등의 예외 발생 시에는 무조건 차단하는 Fail-Closed 정책을 적용하여 보안 사고를 원천 방지했습니다."

### 슬라이드 7: 핵심 검열 기술 구조: 추론(GenMask)
> "Phase 3의 생성형 마스킹 로직입니다. 소재, 공정 파라미터 등 6종의 기밀 카테고리를 추출하되, 과마스킹을 막기 위해 날짜나 일반 수량은 네거티브 리스트로 제외했습니다."
> "아래 코드를 보시면 이 Phase 2와 Phase 3는 `asyncio.gather`를 통해 동시에 실행됩니다. 하나의 프롬프트에 두 가지 지시를 내리면 모델이 헷갈리는 Task Confusion을 방지하기 위해 작업을 분리하고, 병렬로 묶어 시간 지연을 절반으로 줄였습니다."

### 슬라이드 8: PoC 구현 환경 + 시행착오 타임라인
> "저희는 가상 기업 오토코어의 환경을 가정하여, GPU가 없는 인텔 울트라 7, 32GB 램의 사내 업무용 노트북에서 시제품을 구동했습니다."
> "이 과정에서 정규식 전용이던 V1부터 경량 AI 모델을 거쳐, 최종적으로 아시아 언어 추론에 강한 Qwen2.5 7B 모델을 적용한 V4 하이브리드까지 아키텍처를 진화시켰고, 방어율을 26%에서 78%까지 끌어올렸습니다."

### 슬라이드 9: 핵심 기능 구현 검증 (시연 요약)
> "실제로 구현된 시스템의 동작을 화면으로 검증하겠습니다. 정상 통과, 마스킹 및 역치환, 인젝션 차단, 그리고 보안 감사 추적의 4가지 주요 기능입니다."

### 슬라이드 10: 핵심 기능 구현 검증 1 - 정상 통과
> "먼저 일반적인 소프트웨어 명명 규칙을 묻는 정상 질의는 게이트웨이를 무사히 통과하여 원활하게 응답을 받아옵니다."

### 슬라이드 11: 핵심 기능 구현 검증 2 - 동적 마스킹 및 역치환
> "핵심 기능인 마스킹과 역치환입니다. 사원번호 EMP-123을 입력하면 우측 터미널 로그에서 보시듯, 게이트웨이가 이를 난수로 치환하여 전송합니다. 외부 AI는 난수 그대로 정상적인 답변을 생성하고, 게이트웨이가 이를 다시 EMP-123으로 역치환하여 직원의 화면에 띄워줍니다."

### 슬라이드 12: 핵심 기능 구현 검증 3 - 프롬프트 인젝션 차단
> "프롬프트 인젝션 공격입니다. '이전 지시를 무시하라'는 우회 시도가 들어오면, 로컬 LLM이 즉각적으로 의도를 파악하고 화면에 붉은색 경고창과 함께 통신을 원천 차단(BLOCKED)합니다."

### 슬라이드 13: 핵심 기능 구현 검증 4 - 보안 감사 추적
> "마지막으로 관리자 대시보드입니다. 허용, 차단, 마스킹된 모든 이벤트의 원본과 치환된 프롬프트가 DB에 영구 기록되어 사후 감사 추적이 가능합니다."
> "지금까지 아키텍처와 시제품 동작을 보여드렸습니다. 이어지는 성능 검증과 한계 분석 파트는 발표자 B가 진행하겠습니다."

## ===== 발표자 B 파트 (슬라이드 14 ~ 22) =====

### 슬라이드 14: 시스템 성능 및 병목(Latency) 최적화
> (발표자 교체) "성능 평가를 맡은 발표자 B입니다. 무거운 LLM 보안 검열을 도입할 때 가장 우려되는 것은 응답 속도입니다. 표를 보시면, 초기 모델 적재 시간을 제외한 웜 상태(Warm State)에서는 일반 질의 기준 평균 1.51초, 기밀 포함 시 2.62초대의 쾌적한 속도를 보장합니다. 이는 앞서 보신 `asyncio.gather` 비동기 병렬 처리 로직을 통해 I/O 대기 시간을 최소화한 아키텍처 최적화의 성과입니다."

### 슬라이드 15: 보안성 (Quality) — 1,200건 테스트 결과
> "저희는 한국어와 영어 총 1,200건의 자동화 QA 데이터셋을 통해 보안성을 검증했습니다."
> "결과를 보시면, 최종 V4 하이브리드 모델은 종합 방어율 78.33%를 기록했으며, 특히 프롬프트 인젝션 차단율은 무려 91.67%를 달성했습니다. 오탐률 역시 1,200건 중 단 1건(0.08%)으로 허용 범위 내로 극도로 통제되었습니다."

### 슬라이드 16: 성과 — 극한의 정밀도 (Precision)
> "특히 자랑스러운 지표는 정밀도입니다. 한국어 테스트 600건에서 오탐이 단 1건도 없는 정밀도 100%를 달성했습니다. 오탐이 없다는 것은 '보안 때문에 정상 업무가 막혀 직원들이 항의하는 일이 제로'라는 뜻입니다. 우리가 설계한 Fail-Closed 정책이 실무에서 매우 안정적으로 동작함을 증명합니다."

### 슬라이드 17: V1 -> V4 아키텍처 진화 비교
> "정규식 단독이었던 V1과 비교하면, 방어율은 26%에서 약 2.9배 향상된 78%를 기록하면서도, 정밀도는 여전히 완벽하게 유지했습니다. 하이브리드 아키텍처가 정답이었음을 데이터가 말해주고 있습니다."

### 슬라이드 18: 한계점 (※심사위원 방어용 핵심 슬라이드)
> "물론 한계도 명확히 존재합니다. 방어율 78.33%는 역으로 말하면 위협의 22%를 놓친다는 뜻입니다."
> "미탐의 주요 원인은 키릴 문자를 섞어 쓰는 호모글리프 공격이나, 한글 초성을 분리하는 교묘한 우회였습니다. 하지만 강조하고 싶은 것은, **이것은 7B라는 소형 모델 추론 능력의 한계이지, 저희가 설계한 하이브리드 아키텍처의 결함이 아니라는 점입니다.**"
> "만약 아키텍처가 잘못되었다면 정밀도 99.86%는 나올 수 없습니다. 이 뼈대의 견고함 위에 모델만 더 큰 것으로 교체하면 방어율은 즉시 상승할 것입니다."

### 슬라이드 19: Defense-in-Depth 논증 (심층 방어)
> "이 시스템은 기존 솔루션을 대체하는 것이 아닙니다. 기존 방화벽이 막아주지 못하는 LLM 프롬프트 인젝션이나 비정형 기밀 유출이라는 '새로운 AI 보안 사각지대'를 완벽히 보완하는, 사내 방화벽 뒤에 추가하는 AI 전용 보안 계층입니다."

### 슬라이드 20: 향후 연구 방향 (2단계 로드맵)
> "향후 연구 방향은 두 가지입니다. 단기적으로는 현재 7B의 한계를 돌파하기 위해 70B 이상의 대형 모델을 적용하되, 양자화 등 로컬 경량화 기술을 묶어 사내망에서 구동하도록 최적화할 것입니다. 장기적으로는 금융, 제조, 의료 등 산업 도메인별 고유 기밀 패턴을 추가로 파인튜닝하여 엔터프라이즈 맞춤형으로 고도화하겠습니다."

### 슬라이드 21: 참고 문헌 및 출처 자료
> "본 연구에 사용된 주요 통계와 OWASP 가이드라인, 기밀 유출 사례, 그리고 활용된 핵심 오픈소스 아키텍처 출처입니다."
